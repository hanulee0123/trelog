# Claude 作業フォルダの使い方

Claude Code 向けの作業メモはタスクごとに個別の Markdown ファイルとして追加してください。共通のテンプレートをコピーして、必要な情報を埋めるだけで新しいタスクメモを作成できます。

## 構成
- `tasks/`: タスクごとのメモを保存するフォルダ。ファイル名は `task-YYYYMMDD-<slug>.md` など日付と内容が分かる形式にしてください。
- `templates/task-template.md`: タスクメモのテンプレート。コピーして利用します。

## 使い方
1. `templates/task-template.md` をコピーして新しいファイルを作成します。
   - 例: `cp claude/templates/task-template.md claude/tasks/task-20240901-stretch-reminder.md`
2. ファイル冒頭のメタ情報（タイトル・担当・期限・状態）を埋め、チェックリストに作業内容を記入します。
3. タスクが終わったら状態を `Done` に更新し、必要に応じて振り返りを追記してください。

## 運用のヒント
- 1 ファイル 1 タスクを基本とし、粒度は 1〜2 日で終わる作業を目安にします。
- 関連タスクがある場合は「関連タスク」欄にファイル名やリンクを書き、依存関係を明示します。
- Pull Request を作成する際は、該当タスクファイルへのリンクを PR 説明に添えると管理しやすくなります。
