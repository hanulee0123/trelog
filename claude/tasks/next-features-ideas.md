# 今後実装予定の機能メモ

このドキュメントは、次回以降の開発で実装を検討している機能のメモです。
優先度や詳細仕様は今後調整します。

## カレンダー機能

### 要件
- トレーニング記録をカレンダー形式で表示
- 各日付にトレーニングの概要を表示（何の種目をしたか）
- 日付をタップすると詳細ビューを表示
  - その日に行った全種目
  - 各種目のセット詳細（重量、回数）
  - インターバル情報
  - メモがあれば表示

### 技術的検討
- カレンダーライブラリ検討
  - `react-calendar` または `date-fns` + 自作
  - モバイルファーストのタッチ操作に対応
- 月表示・週表示の切り替え
- 当日以外の日付での記録追加機能
- 記録がある日のマーキング（ドット表示など）

### UI/UX
- カレンダーグリッドに種目名を簡潔に表示（例: ベンチ、スクワット）
- タップでモーダルまたはスライドアップで詳細表示
- 詳細ビューから編集・削除も可能に

## トレーニング記録フォームの改善

### セット削除機能
- 現状：セットの追加のみ可能
- 要望：セットの削除も可能にする
- 実装案：
  - 各セット行に削除ボタン（×アイコン）を追加
  - 最後の1セットは削除不可にする（最低1セットは必須）
  - 削除時に確認ダイアログは不要（誤操作防止よりもスピード重視）

---

## ストレッチ提案の強化

### 記録ベースのストレッチ提案
- 現状：種目を選択すると固定のストレッチを提案
- 要望：記録したトレーニング内容を参照して、適切なストレッチを自動提案
- 実装案：
  - トレーニング履歴から当日または直近の記録を取得
  - 実施した種目に基づいてストレッチをレコメンド
  - 複数種目を実施している場合は、すべての部位をカバーするストレッチを提案
  - 「今日のトレーニング後におすすめのストレッチ」のような表示

## トレーニング種目の詳細情報

### 種目ごとの解説データ
- 現状：種目名のみで詳細情報なし
- 要望：各種目について以下の情報を提供
  - **効く部位**: 文字とイラストで視覚的に説明（例: 大胸筋、三角筋前部）
  - **ポイント**: トレーニングのコツや注意点を簡潔にまとめる
  - これらはユーザーが設定するものではなく、アプリ側で固定データとして持つ

### 実装案
- 種目マスターデータの作成（`src/data/exercises.ts`）
  ```typescript
  interface ExerciseInfo {
    name: string;
    targetMuscles: string[]; // 主要ターゲット筋肉
    secondaryMuscles?: string[]; // 補助的に効く筋肉
    illustration?: string; // イラストのパス or URL
    tips: string[]; // ポイント・コツ
  }
  ```
- 種目選択時または記録詳細表示時に、この情報を表示
- イラストは将来的に追加（まずは文字のみでも可）
- 情報表示用の専用コンポーネント作成（`ExerciseDetail.tsx`）

### データ例
```typescript
{
  name: 'ベンチプレス',
  targetMuscles: ['大胸筋'],
  secondaryMuscles: ['三角筋前部', '上腕三頭筋'],
  tips: [
    '肩甲骨を寄せて胸を張る',
    'バーは乳首の少し下に下ろす',
    '足を踏ん張って体幹を安定させる'
  ]
}
```
